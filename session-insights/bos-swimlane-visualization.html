Integrated View
SDLC structure with BOS as the observability implementation
DEFINE
IMPLEMENT
OPERATE
PRODUCT
Define what matters
  • Who cares about this service
  • What they expect
  • What breaks if it fails
Decide what's in scope
  • Which services need BOS
Review & approve
  • Do the signals reflect reality?
  • Sign off on dashboards
Answer questions
  • Business context when Dev asks
Tell Ops what matters
  • Incident priority calls
  • Update expectations after incidents
Make the call
  • Go/no-go on escalations
APP DEV
Figure out what data exists
  • Where's the data? (logs, metrics, traces)
  • What's missing?
Say if it's doable
  • Can we measure what Product wants?
Build the queries
  • Write the SPL/SQL
  • Define the signals
  • Add instrumentation where needed
Wire up the data
  • Connect to data sources
Fix data problems
  • Broken queries
  • Missing instrumentation
  • Data source issues
PLATFORM OPS
Set the guardrails
  • SLO targets (what's acceptable)
  • Alert thresholds (when to page)
  • Dashboard views (who sees what)
Build the dashboards & alerts
  • Configure dashboards
  • Set up alert routing
  • Write the runbooks
  • Plug into pipelines
Watch & respond
  • Keep dashboards healthy
  • Respond to alerts
  • Run the playbooks
  • Handle incidents

IMPLEMENT
OPERATE
PRODUCT

"$2.3M daily revenue at risk if credit checks fail"
Impact Category
Customer Experience, Financial
Business Health Target
"≥ 99% success rate during peak hours (9am-5pm)"
Review & approve
Signal Validation
"Confirmed: success_rate signal reflects actual loan officer experience"
Dashboard Sign-off
"L4 Executive view approved for production"
Tell Ops what matters
Incident Priority
"Credit check down = P1 (direct revenue impact)"
Expectation Update
"Lower threshold to 4s based on Q3 incident learnings"
APP DEV
Figure out what data exists
Data Sources
Splunk: credit_check_logs
Prometheus: api_latency_ms
Gap Analysis
"Missing: end-to-end transaction tracing"
Build the queries
Query
SELECT avg(response_time) FROM credit_check WHERE status='success'
Signal Query Logic
credit_check_success_rate = successful_calls / total_calls
Instrumentation
"Added trace_id to credit check API calls"
Fix data problems
Query Fix
"Fixed timezone bug in latency calculation"
Instrumentation Fix
"Re-enabled trace sampling after v2.3 deploy"

Watch & respond
Dashboard Maintenance

"Acknowledged within 5 min SLA"
